2. Introducción a los Modelos de Red: ¿Por Qué los Necesitamos?

Imagina que intentas construir un edificio sin planos, o que diferentes equipos de construcción usan medidas y herramientas incompatibles. Sería un caos, ¿verdad?

En las redes, sucede algo similar. Para que dispositivos de diferentes fabricantes y tecnologías puedan comunicarse entre sí, necesitamos un lenguaje y una estructura estandarizada. Aquí es donde entran los modelos de red.

Término: Modelo de Red

Un modelo de red es un marco conceptual que divide las funciones de comunicación de una red en un conjunto de capas o niveles. Cada capa tiene responsabilidades específicas y se comunica con las capas adyacentes.

Propósitos de los Modelos de Red:

  • Estandarización: Permite que fabricantes de hardware y desarrolladores de software creen productos compatibles.
  • Modularidad: Divide la complejidad de la red en partes más pequeñas y manejables.
  • Interoperabilidad: Asegura que diferentes sistemas puedan comunicarse entre sí.
  • Facilita el Aprendizaje y el Diseño: Proporciona una estructura lógica para entender y diseñar redes.
  • Resolución de Problemas (Troubleshooting): Permite aislar y diagnosticar problemas en una capa específica.

3. El Modelo OSI (Open Systems Interconnection)

El Modelo OSI, desarrollado por la Organización Internacional de Normalización (ISO) en la década de 1980, es un marco de referencia conceptual que describe cómo la información se mueve a través de una red. Aunque el modelo TCP/IP es el que se usa en la práctica, OSI es fundamental para entender los principios de la comunicación en red.

Término: Modelo OSI

Un modelo de 7 capas que describe las funciones necesarias para la comunicación de red, desde la aplicación del usuario hasta el medio físico.

Representación visual del Modelo OSI con sus 7 capas.

Las 7 Capas del Modelo OSI:

Para recordar las capas de abajo hacia arriba, una mnemotécnica común es: "Por Favor No Tires Seis Perros Alguien" o "Aplicaciones Presentan Sesiones de Transporte de Red de Datos Físicos" (de arriba hacia abajo).

  1. Capa 1: Física (Physical Layer)

    Función: Define las especificaciones eléctricas, mecánicas, de procedimiento y funcionales para activar, mantener y desactivar el enlace físico entre sistemas. Se encarga de la transmisión de bits puros sobre un medio de comunicación.

    Ejemplos: Cables (Ethernet, fibra óptica), conectores (RJ45), voltajes, tasas de bits, hubs, repetidores.

    PDU: Bits

  2. Capa 2: Enlace de Datos (Data Link Layer)

    Función: Proporciona un tránsito de datos fiable a través de un enlace físico. Se encarga del direccionamiento físico (MAC), control de flujo y detección/corrección de errores en el nivel de enlace.

    Ejemplos: Direcciones MAC, tramas Ethernet, PPP (Point-to-Point Protocol), switches de capa 2.

    PDU: Trama (Frame)

  3. Capa 3: Red (Network Layer)

    Función: Proporciona servicios para intercambiar segmentos de datos individuales entre dispositivos finales a través de una red. Se encarga del direccionamiento lógico (IP) y el enrutamiento (routing) de paquetes.

    Ejemplos: Direcciones IP (IPv4, IPv6), routers, protocolos de enrutamiento (RIP, OSPF).

    PDU: Paquete (Packet)

  4. Capa 4: Transporte (Transport Layer)

    Función: Segmenta los datos de las aplicaciones para su transporte a través de la red y reensambla los datos recibidos. Proporciona servicios de comunicación de extremo a extremo, incluyendo control de flujo y fiabilidad (orientado a conexión como TCP) o rapidez (sin conexión como UDP).

    Ejemplos: TCP (Transmission Control Protocol), UDP (User Datagram Protocol), números de puerto.

    PDU: Segmento (Segment - TCP), Datagrama (Datagram - UDP)

  5. Capa 5: Sesión (Session Layer)

    Función: Establece, gestiona y termina las sesiones de comunicación entre aplicaciones. Controla el diálogo entre los procesos de las aplicaciones.

    Ejemplos: NetBIOS, RPC (Remote Procedure Call), SQL.

    PDU: Datos

  6. Capa 6: Presentación (Presentation Layer)

    Función: Traduce, comprime y cifra los datos para asegurar que los datos de la capa de aplicación de un sistema puedan ser leídos por la capa de aplicación de otro. Maneja la sintaxis y la semántica de la información.

    Ejemplos: JPEG, MPEG, ASCII, cifrado (SSL/TLS).

    PDU: Datos

  7. Capa 7: Aplicación (Application Layer)

    Función: Proporciona servicios de red directamente a las aplicaciones del usuario. Es la capa más cercana al usuario final.

    Ejemplos: HTTP (navegación web), FTP (transferencia de archivos), SMTP (correo electrónico), DNS (resolución de nombres de dominio).

    PDU: Datos

4. El Modelo TCP/IP (Transmission Control Protocol/Internet Protocol)

El Modelo TCP/IP es el conjunto de protocolos que realmente impulsan Internet y la mayoría de las redes modernas. Fue desarrollado antes que el modelo OSI y es más práctico, consolidando algunas de las capas de OSI.

Término: Modelo TCP/IP

Un modelo de 4 (o 5) capas que define cómo los datos deben ser empaquetados, direccionados, transmitidos, enrutados y recibidos.

Representación visual del Modelo TCP/IP con sus capas.

Las Capas del Modelo TCP/IP:

  1. Capa de Acceso a la Red (Network Access Layer)

    Función: Combina las funciones de las capas Física y de Enlace de Datos del modelo OSI. Se encarga de los detalles de cómo los datos son enviados físicamente a través de la red (hardware, controladores de dispositivo, direcciones MAC).

    Ejemplos: Ethernet, Wi-Fi, controladores de tarjeta de red.

  2. Capa de Internet (Internet Layer)

    Función: Equivalente a la capa de Red del modelo OSI. Se encarga del direccionamiento lógico (IP) y el enrutamiento de paquetes entre diferentes redes.

    Ejemplos: IP (Internet Protocol), ICMP (Internet Control Message Protocol).

  3. Capa de Transporte (Transport Layer)

    Función: Equivalente a la capa de Transporte del modelo OSI. Proporciona comunicación de extremo a extremo entre aplicaciones, incluyendo segmentación, control de flujo y fiabilidad.

    Ejemplos: TCP (Transmission Control Protocol), UDP (User Datagram Protocol).

  4. Capa de Aplicación (Application Layer)

    Función: Combina las funciones de las capas de Sesión, Presentación y Aplicación del modelo OSI. Proporciona servicios de red a las aplicaciones del usuario.

    Ejemplos: HTTP, FTP, SMTP, DNS.

5. Comparación y Relación entre OSI y TCP/IP

Aunque tienen diferente número de capas, sus funciones son muy similares:

Tabla comparativa visual de las capas de OSI y TCP/IP.

  • OSI: Es un modelo conceptual. Es excelente para entender los procesos de red y para el troubleshooting.
  • TCP/IP: Es un modelo funcional y práctico. Es el que se implementa realmente en Internet.

El Proceso de Encapsulación y Desencapsulación (PDU)

Cuando envías un mensaje, tus datos viajan "hacia abajo" por las capas del modelo, y cada capa añade su propia información de control (encabezados). Esto se llama encapsulación. Al llegar al destino, los datos viajan "hacia arriba", y cada capa elimina el encabezado que le corresponde, en un proceso llamado desencapsulación.

PDU (Protocol Data Unit): Es la unidad de datos en cada capa. Por ejemplo:

  • Capa de Aplicación: Datos
  • Capa de Transporte: Segmento (TCP) / Datagrama (UDP)
  • Capa de Red: Paquete
  • Capa de Enlace de Datos: Trama
  • Capa Física: Bits

6. Aplicaciones Prácticas de los Modelos

Conocer estos modelos no es solo teoría; es una herramienta poderosa para entender y solucionar problemas en redes.

  • Diagnóstico de Problemas: Si no puedes acceder a una página web, ¿es un problema de cable (Capa 1), de dirección IP (Capa 3), o del servidor web (Capa 7)? Los modelos te ayudan a acotar el problema.
  • Diseño de Redes: Permiten diseñar redes de manera estructurada, asegurando que los componentes de cada capa sean compatibles.
  • Desarrollo de Aplicaciones: Los desarrolladores saben qué servicios esperar de las capas inferiores.
  • Seguridad: Entender cómo funcionan las capas ayuda a identificar vulnerabilidades y aplicar medidas de seguridad en el nivel correcto.

Ejemplo: Una Petición Web (HTTP)

  1. Capa de Aplicación (OSI L7 / TCP/IP Aplicación): Tu navegador (HTTP) solicita una página web.
  2. Capa de Presentación (OSI L6): Los datos HTML/CSS/JS se preparan para la transmisión.
  3. Capa de Sesión (OSI L5): Se establece una sesión para tu navegación.
  4. Capa de Transporte (OSI L4 / TCP/IP Transporte): Los datos se dividen en segmentos (TCP) y se les asignan números de puerto (ej. 80 para HTTP).
  5. Capa de Red (OSI L3 / TCP/IP Internet): Cada segmento se encapsula en un paquete IP, se le añade la dirección IP de origen y destino.
  6. Capa de Enlace de Datos (OSI L2 / TCP/IP Acceso a la Red): El paquete se encapsula en una trama, se le añade la dirección MAC de origen y destino del siguiente salto.
  7. Capa Física (OSI L1 / TCP/IP Acceso a la Red): La trama se convierte en bits y se envía a través del cable o el aire.

Actividades Interactivas de la Sesión: ¡Jugando con las Capas!

  1. Actividad 1: "Construye tu PDU" en Miro (25 minutos):

    En grupos, utilizarán Miro para simular el proceso de encapsulación y desencapsulación de datos a través de las capas de la red.

    Instrucciones para usar Miro:

    1. El docente proporcionará un enlace a una pizarra de Miro para cada grupo.
    2. En la pizarra, encontrarán elementos predefinidos (o los crearán): "Datos de Aplicación", "Encabezado de Sesión", "Encabezado de Presentación", "Encabezado de Transporte (TCP/UDP)", "Encabezado de Red (IP)", "Encabezado de Enlace de Datos (Ethernet)", y "Bits".
    3. Tarea del Grupo:
      • Encapsulación (Envío): Partiendo de los "Datos de Aplicación", arrastren y coloquen los diferentes "Encabezados" en el orden correcto (de arriba hacia abajo en OSI) para formar un Segmento, luego un Paquete, y finalmente una Trama. Pueden añadir notas adhesivas explicando qué información añade cada encabezado.
      • Desencapsulación (Recepción): Luego, inviertan el proceso. Partiendo de los "Bits" que forman una Trama, eliminen los encabezados en el orden correcto (de abajo hacia arriba) hasta llegar a los "Datos de Aplicación" originales.
    4. Utilicen flechas y texto para indicar el flujo de los datos y las transformaciones en cada capa.
    5. Al finalizar, cada grupo presentará su "Recorrido de la PDU" a la clase.
  2. Actividad 2: "¿Qué Capa Soy?" - Juego de Roles o Tarjetas (20 minutos):

    El docente tendrá tarjetas con descripciones de funciones, dispositivos o protocolos. Los grupos deberán identificar a qué capa del modelo OSI (y su equivalente en TCP/IP) corresponde.

    Ejemplos de Tarjetas:

    • "Soy responsable de las direcciones MAC." (Capa 2: Enlace de Datos)
    • "Me encargo de los cables y los voltajes." (Capa 1: Física)
    • "Soy el protocolo HTTP." (Capa 7: Aplicación)
    • "Mi trabajo es el enrutamiento de paquetes entre redes." (Capa 3: Red)
    • "Me preocupo por la fiabilidad de la entrega de extremo a extremo (TCP)." (Capa 4: Transporte)
    • "Soy un router." (Principalmente Capa 3: Red)
    • "Soy un switch." (Principalmente Capa 2: Enlace de Datos)

7. Conclusión de la Sesión:

Hoy hemos desentrañado los "planos" de las redes: los modelos OSI y TCP/IP. Comprender estas arquitecturas por capas es fundamental para cualquier analista de sistemas, ya que les permite entender cómo se comunican los dispositivos, cómo se organizan los protocolos y, lo que es más importante, cómo diagnosticar y resolver problemas de red de manera metódica.

Aunque el modelo TCP/IP es la base de Internet, el modelo OSI sigue siendo una herramienta conceptual invaluable para el aprendizaje y la resolución de problemas. En la próxima sesión, profundizaremos en los Protocolos y Estándares de Red, que son las reglas específicas que operan dentro de estas capas.
