2. Refresco de Subnetting: La Lógica de la División

El subnetting es el proceso de dividir una red IP grande en subredes más pequeñas y manejables. Esto es crucial por varias razones:

  • Eficiencia del Espacio IP: Reduce el desperdicio de direcciones IP.
  • Rendimiento de la Red: Reduce el tamaño de los dominios de broadcast, lo que mejora el rendimiento.
  • Seguridad: Permite aislar segmentos de red para mayor seguridad.
  • Gestión Simplificada: Facilita la administración de redes grandes.

Conceptos Clave del Subnetting:

  • Dirección de Red: La primera dirección en una subred, donde todos los bits de la porción de host son ceros. No se puede asignar a un host.
  • Dirección de Broadcast: La última dirección en una subred, donde todos los bits de la porción de host son unos. Se utiliza para enviar datos a todos los hosts en esa subred. No se puede asignar a un host.
  • Rango de Hosts Válidos: Las direcciones IP entre la dirección de red y la dirección de broadcast que pueden ser asignadas a dispositivos.
  • Máscara de Subred: Un número de 32 bits que indica qué parte de una dirección IP corresponde a la porción de red y cuál a la porción de host. Se representa con unos para la porción de red y ceros para la porción de host.
  • Notación CIDR (Classless Inter-Domain Routing): Una forma concisa de representar la máscara de subred, indicando el número de bits de red (ej. `/24` para 255.255.255.0).

3. Actividad 1: Ejercicios de Subnetting (Manual)

Vamos a poner a prueba sus habilidades de cálculo de subnetting. Realicen los siguientes ejercicios en sus cuadernos.

Ejercicio 1.1: Subnetting de Clase C

Dada la dirección de red 192.168.10.0/24, se necesitan crear 4 subredes.

Calcula para cada una de las 4 subredes:

  1. La nueva Máscara de Subred en decimal punteado y notación CIDR.
  2. La Dirección de Red (Network ID).
  3. El Rango de Hosts Válidos.
  4. La Dirección de Broadcast.

Solución Ejercicio 1.1:

Necesitamos 4 subredes. Para calcular el número de bits prestados ($2^n \ge \text{número de subredes}$), $2^2 = 4$. Así que necesitamos prestar 2 bits de la porción de host.

Máscara original: /24 (24 bits de red). Nueva máscara: $24 + 2 = \mathbf{/26}$

Explicación de la Máscara 255.255.255.192:

Una máscara de subred `/26` significa que los primeros 26 bits son para la porción de red. Los primeros tres octetos (24 bits) ya están en 255.255.255. Necesitamos 2 bits más del cuarto octeto.

En el cuarto octeto, los 8 bits tienen los siguientes valores posicionales (de izquierda a derecha):

Posición de bit:  1   2   3   4   5   6   7   8
Valor decimal:   128 64  32  16   8   4   2   1

Si tomamos 2 bits de la porción de host y los convertimos a bits de red, los dos bits más a la izquierda del cuarto octeto se "encienden" (se ponen en 1):

Bits en el cuarto octeto: 1  1  0  0  0  0  0  0
Valores decimales sumados: 128 + 64 = 192

Por lo tanto, la máscara en decimal es 255.255.255.192.

Explicación del Salto de Bloque (Block Size):

El salto de bloque es el tamaño de cada subred. Se calcula tomando `256` y restándole el valor decimal del octeto donde la máscara de subred cambió (en este caso, 192 en el último octeto).

Salto de Bloque = $256 - 192 = \mathbf{64}$.

Esto significa que las direcciones de red de cada subred aumentarán en múltiplos de 64.

Subred 1:

  • Dirección de Red: 192.168.10.0
  • Rango de Hosts Válidos: 192.168.10.1 a 192.168.10.62
  • Dirección de Broadcast: 192.168.10.63

Subred 2:

  • Dirección de Red: 192.168.10.64
  • Rango de Hosts Válidos: 192.168.10.65 a 192.168.10.126
  • Dirección de Broadcast: 192.168.10.127

Subred 3:

  • Dirección de Red: 192.168.10.128
  • Rango de Hosts Válidos: 192.168.10.129 a 192.168.10.190
  • Dirección de Broadcast: 192.168.10.191

Subred 4:

  • Dirección de Red: 192.168.10.192
  • Rango de Hosts Válidos: 192.168.10.193 a 192.168.10.254
  • Dirección de Broadcast: 192.168.10.255

4. Actividad 2: Diseño de Esquema de Direccionamiento IP para una Red Pequeña

Ahora, aplicaremos el subnetting a un escenario más realista. Diseñaremos el esquema de direccionamiento para una pequeña empresa.

Escenario: Red de la Empresa "TechSolutions"

La empresa "TechSolutions" ha adquirido la dirección de red pública 203.0.113.0/24 y necesita dividirla en varias subredes para sus departamentos:

  • Administración: 50 hosts
  • Ventas: 30 hosts
  • Soporte Técnico: 20 hosts
  • Servidores (Web, Correo): 10 hosts (asignación estática)
  • Wi-Fi Invitados: 60 hosts
  • Enlaces WAN: 2 enlaces punto a punto (2 hosts por enlace)

Tarea: Diseña un esquema de direccionamiento IP utilizando VLSM (Variable Length Subnet Masking) para optimizar el uso de direcciones. Asigna las subredes de mayor a menor requisito de hosts.

Para cada subred, determina:

  1. El nombre del segmento.
  2. La Máscara de Subred en notación CIDR y decimal punteado.
  3. La Dirección de Red (Network ID).
  4. El Rango de Hosts Válidos.
  5. La Dirección de Broadcast.
  6. La Dirección IP del Gateway (primera IP válida del rango de hosts).

Solución de Diseño (Ejemplo):

Dirección de red inicial: 203.0.113.0/24

Requisitos de hosts (ordenados de mayor a menor):

  • Wi-Fi Invitados: 60 hosts ($2^6 - 2 = 62$ hosts válidos, máscara /26)
  • Administración: 50 hosts ($2^6 - 2 = 62$ hosts válidos, máscara /26)
  • Ventas: 30 hosts ($2^5 - 2 = 30$ hosts válidos, máscara /27)
  • Soporte Técnico: 20 hosts ($2^5 - 2 = 30$ hosts válidos, máscara /27)
  • Servidores: 10 hosts ($2^4 - 2 = 14$ hosts válidos, máscara /28)
  • Enlaces WAN: 2 hosts ($2^2 - 2 = 2$ hosts válidos, máscara /30)

Esquema de Direccionamiento Propuesto:

  • Wi-Fi Invitados:
    • Máscara: /26 (255.255.255.192)
    • Red: 203.0.113.0
    • Hosts: 203.0.113.1 - 203.0.113.62
    • Broadcast: 203.0.113.63
    • Gateway: 203.0.113.1
  • Administración:
    • Máscara: /26 (255.255.255.192)
    • Red: 203.0.113.64
    • Hosts: 203.0.113.65 - 203.0.113.126
    • Broadcast: 203.0.113.127
    • Gateway: 203.0.113.65
  • Ventas:
    • Máscara: /27 (255.255.255.224)
    • Red: 203.0.113.128
    • Hosts: 203.0.113.129 - 203.0.113.158
    • Broadcast: 203.0.113.159
    • Gateway: 203.0.113.129
  • Soporte Técnico:
    • Máscara: /27 (255.255.255.224)
    • Red: 203.0.113.160
    • Hosts: 203.0.113.161 - 203.0.113.190
    • Broadcast: 203.0.113.191
    • Gateway: 203.0.113.161
  • Servidores:
    • Máscara: /28 (255.255.255.240)
    • Red: 203.0.113.192
    • Hosts: 203.0.113.193 - 203.0.113.206
    • Broadcast: 203.0.113.207
    • Gateway: 203.0.113.193
  • Enlace WAN 1 (Router a ISP):
    • Máscara: /30 (255.255.255.252)
    • Red: 203.0.113.208
    • Hosts: 203.0.113.209 - 203.0.113.210
    • Broadcast: 203.0.113.211
    • IPs a usar: Router: 203.0.113.209, ISP: 203.0.113.210
  • Enlace WAN 2 (Router a Sucursal/Otra Red):
    • Máscara: /30 (255.255.255.252)
    • Red: 203.0.113.212
    • Hosts: 203.0.113.213 - 203.0.113.214
    • Broadcast: 203.0.113.215
    • IPs a usar: Router: 203.0.113.213, Sucursal: 203.0.113.214

5. Actividad 3: Implementación en Cisco Packet Tracer (Configuración GUI)

Ahora, vamos a llevar nuestro diseño a la práctica. Necesitarán tener Cisco Packet Tracer instalado. Si no lo tienen, pueden descargarlo desde la página de Cisco Networking Academy (requiere una cuenta gratuita).

Cisco Packet Tracer: Su Laboratorio Virtual

Cisco Packet Tracer es una herramienta de simulación de red que les permite diseñar, configurar y solucionar problemas de redes sin necesidad de hardware físico. Es invaluable para aprender conceptos de redes de forma práctica.

Escenario a Implementar en Packet Tracer: Conectando Dos LANs con DHCP

Crearemos una topología con dos redes LAN separadas, cada una con su propio servidor DHCP, conectadas por un router.

Esquema de Direccionamiento para este Ejercicio:

  • LAN 1 (Administración):
    • Red: 192.168.1.0/24
    • Gateway (Router): 192.168.1.1
    • Servidor DHCP: 192.168.1.2
    • Rango DHCP: 192.168.1.10 a 192.168.1.254
  • LAN 2 (Ventas):
    • Red: 192.168.2.0/24
    • Gateway (Router): 192.168.2.1
    • Servidor DHCP: 192.168.2.2
    • Rango DHCP: 192.168.2.10 a 192.168.2.254

Dispositivos Necesarios:

  • 1 Router (ej. 1941 o 2911)
  • 2 Switches (ej. 2960-24TT)
  • 4 PCs (PC-PT) - 2 por LAN
  • 2 Servidores (Server-PT) - 1 por LAN, para DHCP

Pasos Detallados para la Configuración (¡TODO GUI!):

Paso 1: Construcción de la Topología Física

  1. Abra Cisco Packet Tracer.
  2. En la barra inferior izquierda, haga clic en "Network Devices" (Dispositivos de Red), luego en "Routers". Arrastre un Router 1941 (o 2911) al área de trabajo.
  3. Haga clic en "Switches". Arrastre dos Switch 2960-24TT al área de trabajo.
  4. Haga clic en "End Devices" (Dispositivos Finales). Arrastre cuatro PC y dos Server al área de trabajo.
  5. Organice los dispositivos: Coloque el Router en el centro. A la izquierda, un Switch, dos PCs y un Server (para LAN 1). A la derecha, el otro Switch, dos PCs y el otro Server (para LAN 2).
  6. Conecte los dispositivos: Haga clic en el icono de "Connections" (Conexiones) - el rayo.
    • Seleccione "Copper Straight-Through" (Cable de Cobre Directo).
    • Conecte Router (GigabitEthernet0/0) a Switch1 (FastEthernet0/1).
    • Conecte Router (GigabitEthernet0/1) a Switch2 (FastEthernet0/1).
    • Conecte PC0 a Switch1 (FastEthernet0/2).
    • Conecte PC1 a Switch1 (FastEthernet0/3).
    • Conecte Server0 (DHCP LAN 1) a Switch1 (FastEthernet0/4).
    • Conecte PC2 a Switch2 (FastEthernet0/2).
    • Conecte PC3 a Switch2 (FastEthernet0/3).
    • Conecte Server1 (DHCP LAN 2) a Switch2 (FastEthernet0/4).
  7. ¡Observe los indicadores de enlace! Los círculos al lado de las conexiones se pondrán verdes cuando los enlaces estén activos (puede tardar unos segundos).

Paso 2: Configuración de las Interfaces del Router (Gateways)

Configuraremos las interfaces del router para que actúen como el Gateway (puerta de enlace) para cada LAN.

  1. Haga clic en el Router.
  2. Vaya a la pestaña "Config" (Configuración).
  3. En el menú de la izquierda, haga clic en "Interface" (Interfaz).
  4. Seleccione la interfaz "GigabitEthernet0/0" (para LAN 1 - Administración).
    • Marque la casilla "Port Status" (Estado del Puerto) a "On" (Encendido).
    • En el campo "IP Address" (Dirección IP), escriba 192.168.1.1.
    • En el campo "Subnet Mask" (Máscara de Subred), escriba 255.255.255.0.
  5. En el menú de la izquierda, seleccione la interfaz "GigabitEthernet0/1" (para LAN 2 - Ventas).
    • Marque la casilla "Port Status" a "On".
    • En el campo "IP Address", escriba 192.168.2.1.
    • En el campo "Subnet Mask", escriba 255.255.255.0.
  6. Cierre la ventana del Router. Observe que los enlaces del router a los switches se pondrán verdes.

Paso 3: Configuración de los Servidores DHCP (GUI)

Cada servidor proporcionará direcciones IP dinámicas a los dispositivos de su respectiva LAN.

Configuración de Server0 (DHCP para LAN 1 - Administración)

  1. Haga clic en Server0.
  2. Vaya a la pestaña "Desktop" (Escritorio).
  3. Haga clic en "IP Configuration" (Configuración IP).
    • Seleccione "Static" (Estático).
    • En "IP Address", escriba 192.168.1.2.
    • En "Subnet Mask", escriba 255.255.255.0.
    • En "Default Gateway", escriba 192.168.1.1 (la IP de la interfaz del router para esta LAN).
    • En "DNS Server", puede escribir 192.168.1.2 (el propio servidor si va a ofrecer servicios DNS) o 8.8.8.8 (DNS público de Google). Para este ejercicio, use 192.168.1.2.
  4. Ahora, vaya a la pestaña "Services" (Servicios).
  5. En el menú de la izquierda, haga clic en "DHCP".
    • Asegúrese de que el servicio "On" (Encendido) esté seleccionado.
    • En "Pool Name", escriba `LAN1-ADMIN` (o cualquier nombre descriptivo).
    • En "Default Gateway", escriba 192.168.1.1.
    • En "DNS Server", escriba 192.168.1.2.
    • En "Start IP Address" (Dirección IP de Inicio), escriba 192.168.1.10 (para que las primeras IPs queden libres para dispositivos estáticos).
    • En "Subnet Mask", escriba 255.255.255.0.
    • En "Maximum Number of Users" (Número Máximo de Usuarios), puede dejar el valor por defecto o ajustarlo (ej. 245 para esta subred).
    • Haga clic en el botón "Save" (Guardar) para guardar la configuración del pool.
  6. Cierre la ventana de Server0.

Configuración de Server1 (DHCP para LAN 2 - Ventas)

  1. Haga clic en Server1.
  2. Vaya a la pestaña "Desktop".
  3. Haga clic en "IP Configuration".
    • Seleccione "Static".
    • En "IP Address", escriba 192.168.2.2.
    • En "Subnet Mask", escriba 255.255.255.0.
    • En "Default Gateway", escriba 192.168.2.1 (la IP de la interfaz del router para esta LAN).
    • En "DNS Server", escriba 192.168.2.2.
  4. Ahora, vaya a la pestaña "Services".
  5. En el menú de la izquierda, haga clic en "DHCP".
    • Asegúrese de que el servicio "On" esté seleccionado.
    • En "Pool Name", escriba `LAN2-VENTAS`.
    • En "Default Gateway", escriba 192.168.2.1.
    • En "DNS Server", escriba 192.168.2.2.
    • En "Start IP Address", escriba 192.168.2.10.
    • En "Subnet Mask", escriba 255.255.255.0.
    • Haga clic en el botón "Save" para guardar la configuración del pool.
  6. Cierre la ventana de Server1.

Paso 4: Configuración de PCs (Clientes DHCP)

Ahora, configuraremos las PCs para que obtengan sus direcciones IP automáticamente de los servidores DHCP.

  1. PC0 (LAN 1):
    • Haga clic en PC0.
    • Vaya a la pestaña "Desktop".
    • Haga clic en "IP Configuration".
    • Seleccione la opción "DHCP".
    • Observe cómo la PC solicita una dirección IP y la recibe del Server0. Verifique que haya recibido una IP en el rango 192.168.1.x (ej. 192.168.1.10), con Máscara 255.255.255.0, Gateway 192.168.1.1 y DNS 192.168.1.2.
  2. PC1 (LAN 1):
    • Repita los pasos anteriores para PC1. Debería obtener la siguiente IP disponible del pool de LAN 1 (ej. 192.168.1.11).
  3. PC2 (LAN 2):
    • Haga clic en PC2.
    • Vaya a la pestaña "Desktop".
    • Haga clic en "IP Configuration".
    • Seleccione la opción "DHCP".
    • Debería recibir una IP en el rango 192.168.2.x (ej. 192.168.2.10), con Máscara 255.255.255.0, Gateway 192.168.2.1 y DNS 192.168.2.2.
  4. PC3 (LAN 2):
    • Repita los pasos anteriores para PC3. Debería obtener la siguiente IP disponible del pool de LAN 2 (ej. 192.168.2.11).

Paso 5: Verificación de Conectividad

Es crucial verificar que todos los dispositivos puedan comunicarse entre sí.

  1. Verificar IP en PCs:
    • En cualquier PC, vaya a "Desktop" > "Command Prompt".
    • Escriba `ipconfig` y presione Enter. Verifique que la IP, máscara, gateway y DNS sean correctos.
  2. Ping dentro de la misma LAN (LAN 1):
    • Desde PC0, vaya a "Desktop" > "Command Prompt".
    • Haga ping a PC1: `ping 192.168.1.11` (o la IP que haya recibido PC1).
    • Deberían ver respuestas (`Reply from...`).
    • Haga ping a Server0: `ping 192.168.1.2`. Deberían ver respuestas.
  3. Ping a Gateway (desde cualquier PC):
    • Desde PC0, haga ping a su gateway: `ping 192.168.1.1`.
    • Desde PC2, haga ping a su gateway: `ping 192.168.2.1`.
    • Deberían ver respuestas.
  4. Ping entre LANs (a través del Router):
    • Desde PC0 (LAN 1), haga ping a PC2 (LAN 2): `ping 192.168.2.10` (o la IP de PC2).
    • Desde PC1 (LAN 1), haga ping a Server1 (LAN 2): `ping 192.168.2.2`.
    • Deberían ver respuestas. Esto confirma que el router está enrutando correctamente entre las subredes.
  5. Modo de Simulación (Opcional, para visualizar el flujo):
    • En la parte inferior derecha de Packet Tracer, haga clic en la pestaña "Simulation" (Simulación).
    • Haga clic en "Add Simple PDU" (Añadir PDU Simple) - el icono del sobre.
    • Haga clic en una PC de LAN 1 y luego en una PC de LAN 2.
    • Haga clic en el botón "Auto Capture / Play" (Captura Automática / Reproducir) para ver cómo el paquete viaja a través de la red, pasando por el switch, el router y el otro switch hasta el destino.
